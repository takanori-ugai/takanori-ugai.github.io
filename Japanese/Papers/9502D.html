概要 : 本稿では, ソフトウェア群をパッケージを単位として管理し, ユーザ側や開発側か らのアップデート要求に応じてパッケージを配布する枠組とシステムを提案す る. 本稿で提案する枠組は, 顧客側と開発側の両側からの要求に応じ, オンライ ン接続されない配布先もオンライン接続された配布先と同様に支援する. さらに, 顧客側の状況に応じた配布頻度などの設定が可能で, ソフトウェア群の 仕様書からパッケージ間の依存関係などを生成し, 配布量の最適化などを行な う. ソフトウェアは, 動作環境として他のソフトウェアが必須であるとか, 存在し てはいけない等の依存関係をAND-OR グラフを拡張した情報とともに管理され, 配 布選択の基礎情報となる. ソフトウェア配布エージェントは, ソフトウェア群デ ータベースの情報や指示にしたがってソフトウェアの配布やアップデートの情報の 配布を行なう.

abstract : In this paper, we propose an intelligent software package distributing system, which receives requirements from both developers and customers. The system provides services to customers whether they connect with developer through network or not. Furthermore the system controls software package distribution in according to customer's requirements and optimizes the procedure for the distribution. The system consists on the software database and the software distribution agents The software database keeps the dependency information among the software packages. The information are represented by the extended AND-OR graph. The software distribution agents distribute softwares and the information in according with the software packages database.
